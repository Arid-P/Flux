**Version:** 1.0

**Status:** Locked  
**Author:** Ari

## 1. Overview

Data Export allows the user to export tasks, notes, folders, and lists from FluxDone as a structured JSON file within a user-defined date range. The exported file follows the same schema as the Import format (FD P3 F3) and includes a human-readable comment header explaining the structure — both for humans unfamiliar with JSON and for AI assistants reading the file.

## 3. Export Scope

### 3.1 What Gets Exported

| Entity | Export Rule |
|---|---|
| Tasks | All tasks with `date` within the selected date range |
| Subtasks | All subtasks belonging to exported tasks |
| Notes | All notes with `date` within the selected date range |
| Lists | Only lists that are parent of at least one exported task or note |
| Folders | Only folders that are parent of at least one exported list |
| Sections | Only sections that contain at least one exported task or note |
| Completed tasks | ✅ Included if date falls in range |
| Trashed tasks | User toggle (see §5) |
| Tasks with no date | ❌ Excluded — no date = no range membership |

### 3.2 What Is Never Exported

- Habits and habit completion history (separate feature, separate schema)
- App settings and preferences
- Theme configurations
- Widget settings
- Smart list configurations (these are system-defined)

---

## 4. Export JSON Format

Uses the exact same schema as FD P3 F3 (Import). The `_comment` field at the top of the file is always present and contains:

```json
"_comment": "FluxDone Export File — generated by FluxDone on [date]. This file contains your task data exported between [start_date] and [end_date]. FORMAT GUIDE: 'folders' contains your folder structure. Each folder contains 'lists'. Each list contains 'tasks' and 'notes'. Tasks contain 'subtasks'. All dates are YYYY-MM-DD. Times are HH:mm (24-hour format). Priority: 0=none, 1=low, 2=medium, 3=high. This file can be re-imported into FluxDone or read by an AI assistant to understand your task history and schedule."
```

The `exported_at` field is set to the current UTC datetime at the time of export.

## 5. Export Configuration

### 5.1 Export Screen Layout

**Route:** `/settings/data/export`  
**Access:** Settings → Data → Export

```
─────────────────────────────────
  Export Data

  Date range
  [25 Nov 2025]  →  [05 Jan 2026]

  Options
  ☑ Include completed tasks
  ☑ Include notes
  ☐ Include trashed tasks

  Preview
  Estimated: ~847 tasks, 14 notes
  across 12 lists in 3 folders

  [Export →]
─────────────────────────────────
```

### 5.2 Date Range Picker

- Two date pickers: start date and end date
- Default: last 30 days (start = today − 30 days, end = today)
- No maximum range restriction
- Start date must be ≤ end date (enforced)

### 5.3 Options

| Option | Default | Description |
|---|---|---|
| Include completed tasks | ON | Completed tasks with date in range are included |
| Include notes | ON | Notes with date in range are included |
| Include trashed tasks | OFF | Soft-deleted tasks with date in range optionally included |

### 5.4 Preview Count

Before exporting, FD runs a lightweight COUNT query and shows an estimate:
- "Estimated: ~[N] tasks, [M] notes across [X] lists in [Y] folders"
- Updates live as user changes date range or toggles options
- Debounced 500ms to avoid query on every date picker tick

## 6. Export Execution

### 6.1 On Tap "Export"

1. FD runs the full export query (SQLite)
2. Builds the JSON structure in memory
3. Writes to a temp file in app cache directory
4. Android share sheet opens — user can save to Downloads, Drive, share via any app

### 6.2 File Naming

Default filename: `fluxdone_export_[startDate]_to_[endDate].json`  
Example: `fluxdone_export_2025-11-25_to_2026-01-05.json`

### 6.3 Large Export Handling

- Exports up to ~10,000 tasks: handled in memory, instant
- Exports > 10,000 tasks: progress indicator shown during JSON build
- No hard cap on export size — if device runs out of memory, Android handles the OOM gracefully

### 6.4 Export Query

```sql
-- Tasks in range
SELECT t.*, l.name as list_name, l.color as list_color,
       f.name as folder_name, s.name as section_name
FROM tasks t
JOIN lists l ON t.list_id = l.id
LEFT JOIN folders f ON l.folder_id = f.id
LEFT JOIN sections s ON t.section_id = s.id
WHERE t.date >= :startDate 
  AND t.date <= :endDate
  AND t.is_trashed = :includeTrashed  -- 0 if toggle off, both 0 and 1 if toggle on
ORDER BY f.sort_order, l.sort_order, t.date, t.created_at

-- Notes in range (if toggle on)
SELECT n.*, l.name as list_name, f.name as folder_name
FROM notes n
JOIN lists l ON n.list_id = l.id
LEFT JOIN folders f ON l.folder_id = f.id
WHERE n.date >= :startDate
  AND n.date <= :endDate
  AND n.is_trashed = 0
ORDER BY n.date

-- Subtasks for exported tasks
SELECT * FROM subtasks 
WHERE task_id IN (:exportedTaskIds)
ORDER BY sort_order
```

## 7. Relationship to Google Drive Backup (SCR-13)

Export and Google Drive Backup are distinct features:

| Feature | Purpose | Format | Scope |
|---|---|---|---|
| Export (this feature) | Data portability, sharing, AI reading | Human-readable JSON | User-defined date range |
| Google Drive Backup (SCR-13, P2) | Full app backup/restore | Raw SQLite database file | Entire database |

They are accessed from different places:
- Export: Settings → Data → Export
- Google Drive Backup: Settings → Google → Google Drive Backup (SCR-13)
