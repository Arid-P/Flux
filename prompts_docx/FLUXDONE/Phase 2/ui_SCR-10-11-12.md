# ui_SCR-10_folder-management.md
## FluxDone вҖ” Folder Management
**Screen ID:** SCR-10
**Version:** 1.0
**Status:** Locked
**Last Updated:** March 2026

---

## 1. Screen Purpose

SCR-10 is not a standalone full screen вҖ” folder management is handled **inline within the side drawer (SCR-08)** via long-press context menus and dialogs. There is no separate navigation route for folder management. All folder operations (create, rename, reorder, change color, delete) are performed directly in the drawer without leaving the current screen.

---

## 2. Entry Points

All folder management actions are accessed by **long pressing a folder row** in SCR-08, which opens a context bottom sheet.

---

## 3. Create New Folder

**Trigger:** "Add List or Folder" button at bottom of drawer вҶ’ "New Folder" option

**Component:** `AlertDialog`
**Title:** "New Folder"
**Content:** `TextField`, placeholder "Folder name", autofocused, single line, max 50 characters
**Actions:**
- "Cancel" вҖ” dismisses, no action
- "Create" вҖ” validates non-empty, creates folder in database with `sort_order` = last position, dismisses dialog

**On creation:** New folder row appears at the bottom of the Lists section with a fade-in animation (200ms `easeOut`). Folder is collapsed by default.

---

## 4. Rename Folder

**Trigger:** Long press folder row вҶ’ context bottom sheet вҶ’ "Rename folder"

**Component:** `AlertDialog`
**Title:** "Rename folder"
**Content:** `TextField` pre-filled with current folder name, autofocused, all text selected
**Actions:**
- "Cancel" вҖ” dismisses, no change
- "Save" вҖ” validates non-empty, updates `folders.name` in database, updates row label in place

---

## 5. Change Folder Color

**Note:** Folders do not have a color in the FD data schema (`folders` table has no `color_hex` column). Folder rows in the drawer use the default icon color. **This action is not applicable to folders.** Only Lists have colors.

If this requirement changes in V2, a `color_hex` column must be added to the `folders` table and the color picker bottom sheet (defined in SCR-08 Section 14) must be wired to folders.

---

## 6. Reorder Folders

**Trigger:** Long press folder row вҶ’ context bottom sheet вҶ’ "Reorder"

**Behavior:** Activates drag-to-reorder mode on all folder rows in the drawer (see SCR-08 Section 12). Lists nested inside folders are not reordered вҖ” only the folder positions change. `folders.sort_order` is updated in database on drop.

---

## 7. Delete Folder

**Trigger:** Long press folder row вҶ’ context bottom sheet вҶ’ "Delete folder"

**Behavior:** Shows confirmation `AlertDialog` (see SCR-08 Section 15).

**On confirm:**
- All lists inside the folder are deleted (`ON DELETE CASCADE` via `task_lists.folder_id`)
- All tasks inside those lists are deleted (`ON DELETE CASCADE` via `tasks.list_id`)
- Folder row animates out of drawer (fade + height collapse, 250ms)

---

## 8. Flutter Component Mapping

| UI Element | Flutter Widget |
|---|---|
| Create dialog | `showDialog` + `AlertDialog` + `TextField` |
| Rename dialog | `showDialog` + `AlertDialog` + `TextField` |
| Reorder mode | `ReorderableListView` in drawer |
| Delete dialog | `showDialog` + `AlertDialog` |
| Fade-in new row | `AnimatedOpacity` |

# ui_SCR-11_list-management.md
## FluxDone вҖ” List Management
**Screen ID:** SCR-11
**Version:** 1.0
**Status:** Locked
**Last Updated:** March 2026

---

## 1. Screen Purpose

SCR-11 covers all list management operations in FluxDone. Like folder management, list management is handled **inline within the side drawer (SCR-08)** вҖ” there is no separate navigation route. All list operations (create, rename, change color, reorder, move to folder, delete) are performed directly in the drawer via long-press context menus and dialogs.

---

## 2. Entry Points

- **Create:** "Add List or Folder" button вҶ’ "New List"
- **All other actions:** Long press a list row in SCR-08 вҶ’ context bottom sheet

---

## 3. Create New List

**Trigger:** "Add List or Folder" button вҶ’ "New List"

**Component:** `showModalBottomSheet` (creation sheet, not a dialog вҖ” needs more fields)
**Corner radius:** 16dp top corners
**Handle bar:** Standard

**Content (top to bottom):**
1. Title: "New List" вҖ” 16sp, semibold, centered, 16dp top padding
2. `TextField` вҖ” "List name", autofocused, single line, max 50 characters, 16dp horizontal padding
3. Color picker row вҖ” horizontal scrollable preset swatches (same 12 presets as SCR-08 Section 14) + hex input button
4. Folder selector row вҖ” "Folder: [name]" with chevron, tapping opens folder picker (see Section 3.1)
5. "Create" button вҖ” full width, 48dp height, app primary color, 8dp corner radius, 16dp horizontal margin

**Default values:**
- Color: first preset (`#2E7D32`) or last used color
- Folder: currently active folder (if a list is open), otherwise first folder

### 3.1 Folder Picker

**Component:** `showModalBottomSheet` (stacked above creation sheet)
**Content:** `ListView` of all folders, each as a `RadioListTile`
**On selection:** Returns to creation sheet with selected folder name updated

**On creation:** New list row appears inside its folder (folder auto-expands if collapsed) with fade-in animation (200ms `easeOut`). `task_lists.sort_order` = last within folder.

---

## 4. Rename List

**Trigger:** Long press list row вҶ’ "Rename list"

**Component:** `AlertDialog`
**Title:** "Rename list"
**Content:** `TextField` pre-filled with current name, autofocused, all text selected
**Actions:** "Cancel", "Save"

---

## 5. Change List Color

**Trigger:** Long press list row вҶ’ "Change color"

**Component:** Color picker bottom sheet (see SCR-08 Section 14)

**On apply:**
- `task_lists.color_hex` updated in database
- List row color swatch updates immediately
- List row left border updates immediately
- All task cards in SCR-01 for this list update their left border and title color
- Calendar task blocks for this list update their fill color
- FAB color in SCR-01 updates if this is the currently viewed list

---

## 6. Reorder Lists

**Trigger:** Long press list row вҶ’ "Reorder"

**Behavior:** Activates drag-to-reorder mode on all list rows within the same folder (see SCR-08 Section 12). Lists cannot be dragged across folder boundaries via reorder вҖ” use "Move to folder" instead. `task_lists.sort_order` updated in database on drop.

---

## 7. Move List to Different Folder

**Trigger:** Long press list row вҶ’ "Move to folder"

**Component:** `showModalBottomSheet`
**Content:** `ListView` of all folders as `RadioListTile` rows. Currently assigned folder is pre-selected.

**On selection:**
- `task_lists.folder_id` updated in database
- List row animates out of current folder position and into new folder (250ms)
- New folder auto-expands if it was collapsed

---

## 8. Delete List

**Trigger:** Long press list row вҶ’ "Delete list"

**Component:** Confirmation `AlertDialog` (see SCR-08 Section 15)

**On confirm:**
- All tasks in the list are soft-deleted (moved to Trash with `is_trashed = 1`, `trashed_at` = now)
- List row animates out of drawer (fade + height collapse, 250ms)
- If the deleted list was currently open in SCR-01, navigate to the "Today" Smart List

---

## 9. Flutter Component Mapping

| UI Element | Flutter Widget |
|---|---|
| Create bottom sheet | `showModalBottomSheet` with custom content |
| Color picker | `showModalBottomSheet` (from SCR-08) |
| Folder picker | `showModalBottomSheet` with `RadioListTile` list |
| Rename dialog | `showDialog` + `AlertDialog` + `TextField` |
| Reorder mode | `ReorderableListView` in drawer |
| Move to folder | `showModalBottomSheet` with `RadioListTile` list |
| Delete dialog | `showDialog` + `AlertDialog` |

# ui_SCR-12_trash.md

**Screen ID:** SCR-12
**Version:** 1.0

## 1. Screen Purpose

SCR-12 displays all soft-deleted tasks. Tasks remain in Trash for 30 days before automatic permanent deletion. The user can restore individual tasks to their original list or permanently delete them. A bulk "Empty Trash" action permanently deletes all trashed tasks at once.

---

## 2. Navigation & Entry Points

- **Entry:** Tapping "Trash" in the Smart Lists section of SCR-08 side drawer
- **Exit:** Android back gesture or hamburger button
- **Transition:** Same as SCR-01 (horizontal slide-in from right)

---

## 4. Top App Bar

**Component:** `AppBar`
**Height:** 56dp
**Title:** "Trash", 18sp, semibold, primary text color
**Left:** Hamburger `IconButton` вҖ” opens SCR-08
**Right:** "Empty Trash" `TextButton` вҖ” red text (`#E53935`), 14sp, medium weight

---

## 5. Task Card in Trash

Identical to SCR-01 task card with these differences:

- **No checkbox** вҖ” replaced by a `Icons.restore_from_trash` icon (20dp, secondary text color) on the left. Tapping restores the task (see Section 7).
- **Left border accent:** List color at 40% opacity (muted вҖ” task is inactive)
- **Title color:** Secondary text color (not list color вҖ” task is inactive)
- **Additional metadata:** "Deleted [N] days ago" shown in metadata row (e.g., "Deleted 3 days ago"), 12sp, secondary text color
- **Expiry indicator:** If task was trashed 25+ days ago, metadata shows "Expires in [N] days" in orange (`#FB8C00`)

**All other metadata** (due date, start time, subtask count, reminder, description preview, priority flag) render identically to SCR-01.

---

## 6. Grouping

Tasks in Trash are displayed as a **flat list** вҖ” no Section grouping (sections are irrelevant for trashed tasks).

Tasks sorted by `trashed_at` descending (most recently deleted first).

---

## 7. Restore Individual Task

**Trigger:** Tap the restore icon (`Icons.restore_from_trash`) on the left of a task card

**Behavior:**
1. Task `is_trashed` set to 0, `trashed_at` set to null in database
2. Task card animates out of Trash list (fade + height collapse, 300ms)
3. Snackbar: "Task restored to [List name]" вҖ” no action button
4. Task reappears in its original list at its original position

**If original list was deleted:** Task is moved to the default list (first list in the first folder). Snackbar reads: "Task restored to [Default list name] вҖ” original list was deleted."

---

## 8. Swipe Actions on Trash Task Cards

### 8.1 Swipe Right вҖ” Restore

**Background revealed:** Green (`#43A047`), `Icons.restore_from_trash` icon (white, 24dp), left-aligned
**On threshold:** Task restored (same as tap restore, Section 7)

### 8.2 Swipe Left вҖ” Permanent Delete

**Background revealed:** Red (`#E53935`), `Icons.delete_forever` icon (white, 24dp), right-aligned
**On full swipe or threshold:** Task permanently deleted from database (hard delete)
**Snackbar:** "Task permanently deleted" вҖ” no undo action

---

## 9. Empty Trash (Bulk Delete)

**Trigger:** "Empty Trash" `TextButton` in top app bar

**Behavior:**
1. Shows confirmation `AlertDialog`:
   - Title: "Empty Trash?"
   - Body: "This will permanently delete all [N] trashed tasks. This cannot be undone."
   - Actions: "Cancel" (dismisses), "Delete all" (red text `#E53935`)
2. On confirm: All tasks with `is_trashed = 1` permanently deleted (hard delete)
3. All task cards animate out simultaneously (staggered fade, 50ms delay per card, 250ms duration each)
4. Screen transitions to empty state

---

## 10. Auto-Purge

Tasks with `trashed_at` older than 30 days are automatically hard-deleted on app launch. This runs silently in the background (`TaskRepository.purgeExpiredTrash()`) with no user-visible notification.

---

## 11. Empty State

When Trash contains no tasks:

**Component:** `Center` widget with `Column`
**Content:**
- `Icons.delete_outline` illustration icon, 64dp, secondary text color at 40% opacity
- "Trash is empty" вҖ” 16sp, medium, secondary text color
- "Deleted tasks are kept here for 30 days" вҖ” 13sp, regular, secondary text color at 70% opacity
- 16dp gap between icon and title, 8dp between title and subtitle

---

## 15. Data Requirements

| Data | Source |
|---|---|
| All trashed tasks | `ITaskRepository.getTrashedTasks()` |

| Days since trashed | Computed: `(now - trashed_at) / 86400000` |
| Days until expiry | Computed: `30 - days_since_trashed` |
